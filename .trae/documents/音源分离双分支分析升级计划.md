# 音源分离 + 人声/伴奏双分支分析升级计划

## Summary
将当前“直接对混音音频做分析”的流程升级为“双分支”：
1) **Demucs** 先做人声/伴奏（及 drums/bass/other）分离；  
2) 对伴奏（`other`/`no_vocals`）做 **HPSS**：`harmonic` 用于和弦识别、`percussive` 用于节拍/速度；  
3) **madmom** 仅吃 `harmonic accompaniment` 做更干净的和弦识别；  
4) **faster-whisper** 仅吃 `vocals` 做歌词识别；  
5) 人声旋律：优先 **Basic Pitch**（Phase 2 可切到 torchcrepe）；  
并在 `/jobs/{id}/result` 增加 `stems / vocal_melody / lyrics` 等字段（**保持现有 API 字段兼容**）。

> 说明：此文档是实现计划，不会立刻执行改代码；等你补充 pipeline 图/更多细节后再进入执行阶段。

def separate_stems(audio_path: str, output_dir: str) -> dict:
    """
    使用 Demucs 做音源分离
    返回：
    {
      "vocals": ".../vocals.wav",
      "drums": ".../drums.wav",

#### 设计细节（可直接实现）
- **优先使用 Demucs**：默认 `htdemucs`（你已确认）  
- **实现方式**：优先采用 demucs CLI 的 Python 入口（如 `demucs.separate.main([...])`），输出目录可控、日志一致；失败抛出 `RuntimeError` 并带 demucs stderr/关键信息。
- **输出文件组织**：`output_dir/demucs/` 下统一落盘（便于调试）。
- **no_vocals**：如果输出有 `drums/bass/other`，用 `soundfile` 读写并做加和导出 `no_vocals.wav`（用于“other or no_vocals stem”的后续选择）。

#### 可配置项（环境变量）
- `DEMUCS_MODEL=htdemucs`
- `DEMUCS_DEVICE=cpu|cuda`（默认：**自动探测**。若 `torch.cuda.is_available()` 则 cuda，否则 cpu）
- `DEMUCS_SHIFTS=1`（速度/质量权衡）
- `DEMUCS_JOBS=1`（并发线程/进程数）

### C. 新增模块 2：二次分离 `audio_preprocess.py`（Phase 1 必做）
**新增文件**：`services/ai/audio_preprocess.py`

实现函数（按你给的签名）：
```py
def extract_harmonic_percussive(audio_path: str) -> dict:
    """
    使用 librosa.effects.hpss 提取 harmonic / percussive 部分
    返回：
    {
      "harmonic_path": "...",
      "percussive_path": "..."
    }
    """
```

#### 设计细节（可直接实现）
- 输入：`other` 或 `no_vocals` stem（由 main.py 决定优先级）
- 使用 `librosa.load(audio_path, sr=SR, mono=True)`  
- 用 `librosa.effects.hpss(y)` 得到 `y_harmonic/y_percussive`  
- 用 `soundfile.write()` 写出 wav（保持 SR 一致）
- **harmonic 用于和弦识别**，percussive 用于节拍（下一步可把 tempo/beat 改为从 percussive 提取）
- 日志：打印输入/输出、SR、时长、耗时

#### 可配置项（环境变量）
- `AUDIO_SR=44100`（统一采样率，便于 downstream）
- `HPSS_MARGIN=3.0`、`HPSS_KERNEL_SIZE=31`

### F. main.py：将 job 主流程接入“分离+双分支”（Phase 1 必做）
**文件**：`services/ai/main.py`

在 `_run_job()` 中插入以下步骤（并保持每步 try/except 降级）：
1) 建立 job 级临时目录（例如 `tempfile.TemporaryDirectory()`），所有中间文件放这里  
2) `separate_stems(audio_path, tmp_dir)` → stems  
3) `extract_harmonic_percussive(stems["other"] or stems["no_vocals"])` → harmonic/percussive  
4) `analysis = analyze_audio_multi(... chord_path=harmonic_path, key_path=harmonic_path, tempo_path=audio_path or percussive_path)`  
5) `lyrics = transcribe_lyrics(stems["vocals"])`  
6) Phase 2：`vocal_melody = extract_vocal_melody(stems["vocals"])`（可先不影响 alphatex 生成；先回传数据）
7) 继续现有 section/alphatex 逻辑（alphatex 仍保持输出）

#### 临时文件清理
- 默认：job 完成后自动清理临时目录
- 提供开关：`KEEP_AI_ARTIFACTS=1` 时保留到 `{AI_STORAGE_DIR}/{job_id}/...`（便于调试）

#### 日志
main.py 每一步打印：
- step 名称 + 输入输出路径
- 耗时（秒）
- 失败时打印异常栈 + 降级策略（例如“Demucs 失败，回退到 mix 做和弦”）
